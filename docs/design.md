# RokoMonitor - 洛克王国世界 PVP 对战辅助工具

## 1. 项目概述

RokoMonitor 是一款面向《洛克王国世界》的 PVP 对战辅助工具，核心目标是帮助玩家在对战中快速获取对方精灵信息，从而做出更优决策。类似宝可梦对战中的"队伍预览 + 技能池查询"辅助。

## 2. 核心功能

| 功能 | 描述 |
|------|------|
| 精灵识别 | 对战时通过屏幕截图 + 文字识别（OCR）识别对方出场精灵 |
| 技能池查询 | 在本地数据库中查询该精灵的全部可选技能 |
| 信息展示 | 将技能信息（属性、威力、效果等）实时展示给用户 |
| 数据录入 | 多模式录入精灵与技能数据，构建本地数据库 |

## 3. 关键流程

```
┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  屏幕截图    │───▶│  精灵识别     │───▶│  数据库查询   │───▶│  信息展示     │
│ Screen Cap  │    │ Identify     │    │ DB Query     │    │ Display      │
└─────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
      │                   │                    │                    │
      ▼                   ▼                    ▼                    ▼
 捕获游戏窗口画面    OCR识别精灵名称文字   查技能表+属性表       悬浮窗/面板渲染
 定时或手动触发      输出精灵名称          返回完整技能池         高亮关键信息
```

### 3.1 屏幕截图

- **触发方式**：用户按快捷键手动触发 / 对战开始时自动持续截图
- **截图范围**：游戏窗口中精灵出场区域
- **输出**：保存为内存中的图像帧，供后续识别使用

### 3.2 精灵识别

- **方案**：基于文字识别（OCR），识别对战中显示的精灵名称文本
- **流程**：
  1. 对截图进行预处理（裁剪精灵名称区域、灰度化、二值化、降噪）
  2. 调用 OCR 引擎识别精灵名称文本
  3. 对识别结果做模糊匹配（纠正常见 OCR 误识别，如"焰"→"焰"）
  4. 输出精灵名称；识别失败或置信度低时提示用户手动输入
- **优势**：无需采集精灵图像模板库，新精灵上线后零维护成本

### 3.3 数据库查询

- **数据来源**：通过数据录入模块构建的本地 SQLite 数据库
- **数据内容**：
  - 精灵基础属性（HP、攻击、防御、速度等）
  - 精灵属性（草、火、水等）
  - 技能池（可用技能列表）
  - 技能详情（属性、威力、命中、PP、附加效果）
- **查询流程**：
  1. 以 OCR 识别出的精灵名称为主键查询精灵表
  2. 关联查询该精灵的全部可选技能
  3. 组装返回完整技能池信息

### 3.4 数据录入

数据库不依赖固定数据源文件，通过录入模块以多种方式构建和维护。

#### 3.4.1 录入模式

| 模式 | 场景 | 流程 |
|------|------|------|
| 爬虫录入 | 批量初始化数据库；游戏更新后增量同步 | 配置目标网站 → 解析页面结构 → 提取精灵/技能数据 → 写入数据库 |
| 手动录入 | 补充/修正单条数据；新精灵上线但爬虫源未更新 | 在 UI 界面填写精灵名称、属性、技能 → 校验 → 写入数据库 |
| 一图流识别 | 快速录入社区分享的配队截图 | 截图/导入图片 → OCR 识别图中精灵名称列表 → 批量匹配数据库 → 缺失项提示手动补充 |

#### 3.4.2 爬虫录入

- **目标**：洛克王国官方攻略站、Wiki、社区数据库等
- **流程**：
  1. 用户配置爬虫源 URL 和解析规则（或使用内置预设）
  2. 爬取精灵列表页和详情页
  3. 解析提取精灵属性、技能池等结构化数据
  4. 去重后写入数据库，已有记录则更新
- **注意**：遵守目标站点的 robots.txt 和访问频率限制

#### 3.4.3 手动录入

- **入口**：工具设置界面 → 数据管理 → 添加精灵/技能
- **表单字段**：
  - 精灵：名称、属性、基础属性值
  - 技能：名称、属性、威力、命中、PP、效果描述
  - 关联：为精灵绑定可选技能
- **校验**：名称唯一性检查、字段完整性检查

#### 3.4.4 一图流配队识别

- **场景**：社区常以图片形式分享配队（一张图包含 6 只精灵名称）
- **流程**：
  1. 用户粘贴截图或导入图片文件
  2. OCR 识别图中所有精灵名称
  3. 逐一到数据库匹配
  4. 已有精灵 → 自动关联到配队记录
  5. 未匹配精灵 → 列出候选项供用户选择，或提示手动录入
- **输出**：配队精灵列表 + 各精灵技能池概览

### 3.5 信息展示

- **展示形式**：桌面悬浮窗（置顶显示，不遮挡对战操作区域）
- **展示内容**：
  - 对方精灵名称 + 属性
  - 技能列表（按属性/威力排序）
  - 每个技能的属性、威力、效果简述
  - 对我方精灵的属性克制提示（可选）
- **交互**：点击技能可查看详细效果描述

## 5. 技术选型

| 模块 | 技术 | 说明 |
|------|------|------|
| 语言 | Python 3.11+ | 整体技术栈 |
| 屏幕截图 | `mss` | 跨平台高性能截图，适合对战中按帧捕获 |
| OCR 引擎 | `PaddleOCR` | 中文识别精度高，推理速度快，社区活跃 |
| OCR 后处理 | `thefuzz` | 对 OCR 结果做模糊匹配纠错（基于编辑距离） |
| 数据库 | `SQLite` + `SQLAlchemy` | SQLite 零部署；SQLAlchemy ORM 统一模型定义与查询 |
| 爬虫录入 | `requests` + `BeautifulSoup4` | 轻量够用，避免 scrapy 过重依赖 |
| 图像预处理 | `OpenCV-Python` | 截图裁剪、灰度化、二值化、降噪 |
| UI 框架 | `PyQt6` | 悬浮窗置顶、富文本展示、表单录入，能力全面 |
| 全局快捷键 | `pynput` | 监听 F2 等快捷键触发截图识别 |
| 配置管理 | `pydantic` + YAML | 类型安全的配置模型，用户可编辑 YAML |
| 打包发布 | `PyInstaller` | 单文件 exe，玩家零配置使用 |
| 测试 | `pytest` | 单元测试与集成测试 |

## 6. 使用关键流程（用户视角）

1. **启动工具** → 程序加载数据库，初始化 OCR 引擎
2. **进入对战** → 打开游戏 PVP 对战界面
3. **触发识别** → 按下快捷键（如 `F2`），工具截图并 OCR 识别对方精灵名称
4. **查看信息** → 悬浮窗显示对方精灵技能池及关键属性
5. **辅助决策** → 根据技能池判断对方可能使用的技能，选择应对策略
6. **切换精灵** → 对方换精灵时，再次按快捷键识别
